Data set summary¶

There are 6,362,620 transactions. We found on duplicates, no missing values.

Out of 6,362,620 records, there are only 8,213 (0.129%) transactions labeled as fraud. Since our goal is to develop fraudulent transaction prediction models, we have really inbalanced data set.

"The challenge of working with imbalanced datasets is that most machine learning techniques will ignore, and in turn have poor performance on, the minority class, although typically it is performance on the minority class that is most important." Especially in our case, our goal is to label the fraudulent transctions, the minority class.

"One approach to addressing imbalanced datasets is to oversample the minority class. The simplest approach involves duplicating examples in the minority class, although these examples don’t add any new information to the model. Instead, new examples can be synthesized from the existing examples. This is a type of data augmentation for the minority class and is referred to as the Synthetic Minority Oversampling Technique, or SMOTE for short."

There are 11 columns in the data set. Since we'd want to build the fraud prediction model, we'll ingore isFlaggedFraud column. isFraud column is our target label. We add two more columns by taking the first character from nameOrig and nameDest and we name them as DestType and OrigType, which indicate if the accounts are merchants or not. We'll explore the rest 11 columns to identify features which can be used in the prediction model.

  • nuemric variables - amount, oldbalanceOrg, newbalanceOrig, oldbalanceDest, newbalanceDest
  • categorical variables - type, nameOrig, nameDest, DestType, OrigType
  • step - it indicates time sequence, we'd want to explore if converting them weekday and hour of the day better serve the model

We found fraudulent transactions gather around week 3 and there is no fraudulent transaction on week 5, 6, 0. Non-fraudulent transactions gather around week 2 and they happen on week 5. All in all, transactions on week 5 tend to be all non-fraudulent.

feature importance¶

We are intersted in differentiating important features from our feature list. We set up the code below to select important features but we run out of time to finish running the code. Our idea is to select the most important feature and possibly build a smaller model (with less variables). We might have a better model.

# Feature Importance
from sklearn.feature_selection import f_classif, f_regression, SelectKBest

xgb_accuracy = []
for k in range(1,len(X_train.columns)):
    selector = SelectKBest(f_classif, k=k)
    selector.fit_transform(X_train, y_train)
    names = X_train.columns.values[selector.get_support()]
    scores = selector.scores_[selector.get_support()]
    names_scores = list(zip(names, scores))
    df_features = pd.DataFrame(data = names_scores, columns=['Feat_names', 'F_Scores'])
    best_Kfeatures = list(df_features.sort_values(['F_Scores', 'Feat_names'], ascending = [False, True])['Feat_names']) 
    xgb.fit(X_train[best_Kfeatures],y_train)    
    xgb_accuracy.append(xgb.score(X_test[best_Kfeatures],y_test))

temp = xgb.feature_importances_
df_imp = pd.DataFrame(temp)
df_imp.columns=["importance"] 
df_imp["labels"] = best_Kfeatures
df_imp.sort_values("importance",ascending=False,inplace=True)
